# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email **security@vibeping.dev** with:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Any suggested fixes

## Response Timeline

- **48 hours**: We acknowledge your report
- **7 days**: Initial assessment and severity classification
- **30 days**: Fix developed and tested
- **Release**: Coordinated disclosure after patch is available

## Scope

In scope:

- SDK code (packages/sdk)
- Dashboard application (apps/dashboard)
- API endpoints
- Database schema and RLS policies
- Authentication and authorization

Out of scope:

- Denial of service attacks
- Social engineering
- Issues in dependencies (report those upstream)
- Issues requiring physical access

## Safe Harbor

We will not pursue legal action against researchers who:

- Act in good faith
- Avoid accessing or modifying other users' data
- Report findings promptly
- Give us reasonable time to fix before public disclosure

## Recognition

We credit security researchers in our CHANGELOG (with permission).
